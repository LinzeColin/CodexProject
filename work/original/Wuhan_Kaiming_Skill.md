name: 武汉开明-智能工业运维助手
description: |
  本技能为武汉开明高科技有限公司定制，利用国内 AI 模型（DeepSeek、千问/达摩院、大模型、豆包等）和公司内部知识库提供一站式工业运维服务。它集合了旋转窑动态监测分析、故障诊断、齿圈修复指导、车削磨削镗孔工艺咨询以及其他金属表面修复与设备改造建议。用户通过本技能输入设备运行数据、故障现象或加工需求，系统将通过 AI 模型生成专业建议，并基于武汉开明“FGM‑KM‑B.U”新材料技术和先进加工设备提供定制解决方案。

inputs:
  - name: 功能类型
    type: string
    description: |
      指定请求的服务功能，可选值包括：
      1. 动态调测窑建议 – 提供旋转窑运行参数分析、实时监测与调整方案；
      2. 回转窑诊断与建议 – 根据故障现象、运行状况诊断原因并给出解决方案；
      3. 齿圈齿面修复 – 提供大齿圈磨损检测、修复工艺及材料建议；
      4. 机械加工服务 – 提供车削、磨削、镗孔等加工工艺咨询、刀具选择与参数设定；
      5. 其他咨询 – 其他设备维护、金属表面修复、新材料应用等开放式问答。
  - name: 输入内容
    type: string
    description: 根据所选功能传入的详细运行数据、故障描述或加工要求，可包含数值、文字描述、图片路径（通过外部链接提供）等。

outputs:
  - name: 解决方案
    type: string
    description: 返回针对输入内容生成的专业建议、调整方案或工艺指导，格式为段落文本，并在需要时列出关键步骤和注意事项。
  - name: 引用说明
    type: string
    description: 系统用于内部调试的引用资料说明或外部链接，不对终端用户展示。

workflow:
  - 步骤 1：技能选择与参数确认
    描述: 根据用户输入的功能类型检查是否在支持范围内；否则提示用户选择正确的服务。
  - 步骤 2：数据解析与预处理
    描述: 对用户提供的运行数据或故障描述进行结构化处理；调用公司内部知识库（包含测量报告、设备说明书、维修案例等）进行初步匹配。
  - 步骤 3：AI 模型调用
    描述: 通过中国 AI API 网关选择合适的模型（DeepSeek、千问/达摩院、豆包等），构建提示词，结合预处理后的数据生成诊断与建议。模型路由依据任务类型自动选择最适合的模型（如技术分析需推理能力、工艺指导需理解工艺流程）。
  - 步骤 4：结果融合与本地知识修订
    描述: 将 AI 模型生成的建议与武汉开明自有技术标准（如 FGM‑KM‑B.U 新材料修复工艺、动态监测算法、磨削设备参数）进行融合和修订，保证结果符合公司经验和行业规范。
  - 步骤 5：输出构建
    描述: 整理最终建议，输出易于理解的中文报告，包含关键步骤、参数表、风险提示和后续跟踪建议；如有需要，附带二维码链接到详细技术资料或监测报告。

limitations:
  - 系统仅提供技术建议，不承担施工风险。具体操作需由专业工程师执行并确认。
  - 输入的数据应为真实且准确的运行数据或故障描述，错误的输入可能导致输出不准确。
  - 当前版本不支持实时上传大文件，请通过外部链接提供图片或文档路径；未来版本将集成文件上传功能。
  - AI 模型的回答受限于模型能力，极端工况或罕见故障可能需要人工专家进一步判断。
