# 武汉开明智能工业运维助手 – 任务清单

此任务清单用于指导开发适用于武汉开明高科技有限公司的技能程序，汇总工作内容、优先级和交付物。任务分为模块开发、数据准备、模型接入、界面设计及测试验收等阶段。

## 一、数据准备与知识库构建

1. **资料整理与分类**
   - 收集公司宣传册、测量报告、技术方案等原始资料，包括本次提供的《旋转窑动态监测技术方案》、公司简介、各类业务介绍等。
   - 将文档内容按业务板块（动态监测、故障诊断、齿圈修复、机械加工、FGM‑KM 新材料应用等）进行分类，创建索引。

2. **知识库结构设计**
   - 设计统一的知识库格式，如 Q&A 列表、流程图、参数表等，方便在技能执行过程中快速检索。
   - 基于关键数据（运行参数范围、设备型号、加工工艺）建立结构化 JSON/数据库，以便程序调用。

3. **文本与多媒体转录**
   - 将宣传册中的图表、设备参数、工艺流程等内容录入电子档；提取相关图片并存放到 `assets/` 目录，以便生成报告时引用。

4. **中文语料清洗与标注**
   - 对文本数据进行必要的清洗（去除无关文字、提炼要点）；根据应用场景标注特征（如故障类型、修复方法）。

## 二、模块开发

1. **动态调测窑建议模块**
   - **输入解析**：支持输入窑体运行参数（温度、转速、中心线偏差、椭圆度等）和测量数据文件路径。
   - **算法实现**：集成公司现有的动态监测算法，根据 IDM 系统、壳体激光扫描仪和轮测仪采集的数据，对支承滚轮与胎圈运行状态进行分析，计算窑体中心线、椭圆度和偏心率【583987538528893†L374-L382】。
   - **结果输出**：结合 3D‑RGB 模型生成可视化参数（可调用第三方绘图库），提供调整建议和调整步骤；支持输出 PDF 报告。

2. **回转窑运行问题诊断与建议模块**
   - **案例库构建**：汇总常见运行故障及原因，如窑体振动、温度异常、偏心运行等；每个案例包含症状、原因分析和处理建议。
   - **故障诊断逻辑**：根据用户描述和输入参数，通过关键词匹配和规则推理确定可能原因；调用 AI 模型获取更深入的原因分析和建议。
   - **输出格式**：提供故障原因、影响部件、处理方法和预防措施列表。

3. **齿圈齿面修复模块**
   - **磨损检测方法**：梳理齿圈磨损、裂纹等检测指标，结合 IDM 系统和激光扫描仪的测量精度（如 ±0.1mm）【583987538528893†L556-L560】。
   - **修复工艺指南**：整合公司 FGM‑KM‑B.U 新材料修复工艺，编写修复流程，包括表面处理、材料配比、焊补或喷涂、后续热处理等，强调新材料能延长零件寿命并降低成本【652285303063645†L68-L120】。
   - **安全提醒**：提示在修复过程中的安全操作规范、注意事项和质量检验标准。

4. **机械加工服务模块**
   - **设备信息接口**：展示公司生产基地的先进设备（大型数控立车、卧车、齿轮滚齿机及 100T 起重机、加热炉等）【652285303063645†L68-L82】。
   - **工艺咨询**：根据客户提供的加工尺寸、材料和精度要求，推荐合理的加工工艺（车削、磨削、镗孔等），包括刀具选择、切削参数、冷却方式。
   - **报价与计划**：提供加工周期和估算费用（可由管理员在后台维护），生成加工计划书。

5. **其他咨询模块**
   - 提供关于金属表面修复、设备升级改造、新材料应用的开放式问答服务；利用 AI 模型结合知识库回答问题。

## 三、AI 模型接入与路由

1. **模型接入**
   - **DeepSeek**：使用 OpenAI 兼容接口，设置 `base_url` 和 API Key，配置 v4-pro 或 v4-flash 等模型，注意并发限制和退役模型提示【583987538528893†L374-L382】。
   - **千问 / DashScope**：通过兼容模式接口调用 Qwen-Plus 或 Qwen-Max，配置 `api_base` 和 `DASHSCOPE_API_KEY`，支持工具调用和函数调用。
   - **豆包（Seed 2.0）**：使用开放的 OpenAI 兼容服务（EvoLink 或官方接口），配置不同规格模型（Lite/Pro）和上下文长度。
   - **其他模型**：后续可接入 Kimi、MiniMax 等兼容接口，按需配置。

2. **路由策略**
   - 根据任务类别自动选择最适合的模型：
     - 技术分析类任务倾向使用推理强、长上下文模型（如 DeepSeek v4 pro）。
     - 问答与对话类任务倾向使用综合能力强、语义理解好的模型（如千问 Qwen-Max）。
     - 简短查询或多轮对话可使用成本较低的模型（如豆包 Lite）。
   - 支持 fallback 机制：若首选模型调用失败，自动切换至其他模型。

3. **安全与限速**
   - 实施 API 调用速率控制和令牌桶算法，避免超额使用。
   - 对用户输入进行敏感词过滤，防止提交不合规请求。
   - 记录调用日志（模型、参数、时间、费用），供运维人员分析。

## 四、界面设计

1. **Web UI 控制台**
   - 提供登陆界面和功能选择菜单，用户选择需要的模块后上传数据或输入描述。
   - 展示查询结果和报告下载链接（PDF 格式），提供历史记录查询功能。

2. **CLI 命令行工具**（可选）
   - 支持通过命令行传入参数调用各模块，方便脚本化集成。

3. **接口服务**
   - 提供 REST API（MCP Server），便于外部应用集成，输出 JSON 结构化结果。

## 五、测试与验证

1. **单元测试**
   - 为各模块编写单元测试，验证输入解析、知识库查询、模型调用、结果融合等关键环节。

2. **人工评估**
   - 依据 Evals 方法，为每个模块设计小规模评测集，包含典型故障、加工需求或监测数据，评估生成答案的准确度和实用性。

3. **性能测试**
   - 监测系统在高并发和长上下文调用情况下的性能和响应时间，优化模型调用和缓存机制。

## 六、部署与运维

1. **环境配置**
   - 配置服务器环境和依赖库；为各 AI 模型存储 API Key 和 Base URL，严格管理密钥权限。

2. **监控与日志**
   - 部署监控系统，监控 API 调用次数、失败率和系统资源消耗；异常时告警。

3. **版本管理**
   - 制定版本发布策略，记录功能变更、修复情况和已知问题；支持回滚。
