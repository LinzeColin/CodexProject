### 2. 需要做什么

- 先人工审核核心画像，避免把短期噪音当成长期身份。
- 沉淀 Run Contract、Task Pack、HANDOFF、验证命令和低上下文工作流。
- 把研究、回测、paper trading、真实下单授权分层，真实资金动作保持 fail-closed。
- 把课程内容沉淀成 Notion 可复制笔记、术语表、框架对比和作业素材库。
- 把学习目标拆成项目、日课、周复盘、月复盘和自动提醒。
- 把宏观思考落到可训练能力：沟通、判断、工程化、商业化、机会识别。
- 先把核心画像和重要中长期候选审核成 active memory，再让未来 agent 默认先读这些资料。

### 3. 需要记得做什么

- 未来 Codex 协作必须高 ROI、低上下文浪费、可验证、可恢复。
- 金融/交易 agent 可以读取记忆和 secret_ref，但不能自动执行真实 broker/资金动作。
- 课程类内容通常是项目/阶段性知识，默认进入中长期或短期，不直接提升为核心画像。
- Notion 是干净笔记本，GitHub 是 durable state，邮件推送用于维持节奏。
- 你追求的是 ROI 最大化、个人能力成长最大化，以及在目标领域达到顶尖水平。
- OpenAIDatabase 是长期记忆数据库，不是一次性报告目录。
- 未来助手必须先遵守已记录的回答规则、安全边界和授权边界。

- personal memory RAG、agent continuity、跨模型 personalization、长期项目复盘工具。

### 6. 个人能力成长信号

- 把 prompt 能力升级为工程化任务拆解、验证、交付和复盘能力。
- 训练 evidence-based decision making，避免情绪化和高风险自动化。
