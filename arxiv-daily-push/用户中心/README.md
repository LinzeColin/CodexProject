# ADP 用户中心

更新时间：2026-06-26 18:07:30 Australia/Sydney

这里是 ADP 在 GitHub 上的中文主入口。你不需要打开本机目录、运行文件、深层治理文件或原始 JSON，也能判断今天是否正常、邮件有没有发、队列里还有什么、下一步该做什么。

## 先看结论

| 问题 | 现在的答案 | 你该怎么处理 |
|---|---|---|
| 今日已发送 / 总应发送 | 2 / 待确认 | 总应发送必须来自当天正式发送计划，不能手写猜数 |
| 邮件发送模板 | [Email V1 学习邮件模板 UI/UX 预览](./邮件模板预览.md) | 先看预览；源码链接只作为审查证据 |
| 截至今日候选池 | 299 条内容记录；当前展示 11 条 | 候选池包含已生成报告、邮件预览和仍待处理内容 |
| 复习、行动、收益是否已可每日查看 | 本地证据已完成，但每日数量还没有自动刷新到 GitHub 用户中心 | 这是下一步应修的可读性缺口 |
| Stage 2 是否已经正式生产通过 | 没有；最终门仍阻断 | 不能宣称正式生产通过或每日运行已验收 |

## 阅读规则

| 规则 | 原因 |
|---|---|
| GitHub 用户中心是主阅读面 | 用户不应该去本机目录找答案 |
| 本机运行文件只做证据 | 本机 JSON、SMTP 报告和候选队列不是日常阅读入口 |
| 先给结论，再给证据 | 避免把 owner 页面写成工程日志 |
| 全局中文优先 | 只有产品名、任务编号、文件名和必要协议名保留原文 |
| 不用用户中心当生产开关 | 发信、改队列、改计划任务、改公共结构都必须另走任务和验收 |
| 具体更新时间必须由脚本写入 | 避免人工手写未来时间或静态时间漂移 |
